# 🔔 לא שכחתי (Never Forgot)

אפליקציית ווב מתקדמת (PWA) לניהול ומעקב אחר אירועים חשובים — ימי הולדת, אזכרות, חתונות, תורים רפואיים ועוד — עם תזכורות, ברכות אוטומטיות, סנכרון יומן וסיוע AI.

🔗 **כתובת האפליקציה:** [barakaflalo.github.io/lo-shachachti](https://barakaflalo.github.io/lo-shachachti)

---

## ✨ פיצ'רים עיקריים

- **ניהול אירועים** — 12 סוגי אירועים (יום הולדת, אזכרה, חתונה, בר/בת מצווה, טיול, תור רפואי ועוד), קבוצות, חזרה שנתית/חד-פעמית
- **5 שפות מלאות** — עברית, אנגלית, רוסית, ספרדית, ערבית (RTL/LTR אוטומטי), כולל ברכות ומדריך
- **ברכות** — 5 וריאנטים מובנים לכל סוג אירוע + ברכות אישיות + ✨ ברכה מותאמת עם AI (OpenAI / Gemini, BYOK)
- **מתנות ותקציב** — תגיות מתנה, מעקב הוצאות, גרף תקציב לפי קבוצה, קישורי חנויות
- **סנכרון יומן גוגל** — חיבור OAuth דו-כיווני, ייבוא/ייצוא אוטומטי, זיהוי סוגי אירועים
- **לוח שנה עברי/גרגוריאני עם חגים** — יהודי, נוצרי, מוסלמי וישראלי, ניתן לבחירה בהגדרות
- **ייבוא/ייצוא** — אנשי קשר (VCF + Contact Picker), ICS, CSV, גיבוי/שחזור JSON, ייבוא מפייסבוק
- **התראות** — OneSignal, תזכורות מרובות לכל אירוע, שעות שקט
- **עיצוב** — 12 ערכות צבע, מצב כהה/בהיר/אוטומטי, אנימציות מעבר, ניווט בהחלקה (לוח שנה + מסכי הגדרות)
- **PWA** — התקנה למסך הבית, עובד אופליין, Service Worker

---

## 📁 מבנה הפרויקט

| קובץ | תיאור |
|---|---|
| `index.html` | כל האפליקציה — HTML, CSS ו-JS בקובץ אחד (ללא תלויות חיצוניות מעבר ל-OneSignal) |
| `manifest.json` | הגדרות PWA (שם, אייקונים, מצב תצוגה) |
| `sw.js` | Service Worker — cache לאייקונים בלבד; קוד ועיצוב נטענים מהרשת תמיד |
| `OneSignalSDKWorker.js` | Service Worker להתראות Push |
| `icon-192.png`, `icon-512.png` | אייקוני האפליקציה |

> **הערה ארכיטקטונית:** האפליקציה נבנתה בכוונה כקובץ `index.html` אחד מאוחד (CSS+JS מוטמעים) כדי למנוע בעיות cache/גרסאות בין קבצים נפרדים. כל עדכון = העלאת `index.html` בלבד (אלא אם צוין אחרת).

## 🛠️ עדכון / פיתוח

1. עורכים את `index.html`
2. מעלים ל-GitHub (Add file → Upload files → Commit)
3. רענון קשיח בדפדפן (Ctrl+Shift+R) אם יש שינוי ב-Service Worker

### סנכרון Google Calendar
החיבור מוגדר עם Client ID קבוע (Google Cloud Project). פרטים מלאים בהיסטוריית הפיתוח. כל עוד הפרויקט במצב "בדיקה" ב-Google Cloud Console, רק משתמשי בדיקה רשומים יכולים להתחבר — לשחרור לכולם יש לבצע "Publish App" ב-OAuth consent screen.

---

## 🔒 פרטיות

- כל הנתונים נשמרים מקומית בדפדפן (`localStorage`) — אין שרת backend
- חיבור Google Calendar הוא ישיר בין המשתמש לגוגל (OAuth), ללא מעבר בשרת
- מפתחות AI (OpenAI/Gemini) נשמרים מקומית בלבד ולא משותפים
